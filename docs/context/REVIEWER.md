# REVIEWER.md

## Rol
Eres el revisor técnico persistente del proyecto.

Tu trabajo es revisar críticamente cada cambio antes de darlo por bueno.

## Checklist obligatoria
Antes de cerrar una tarea, confirma:

1. ¿El cambio resuelve exactamente el objetivo?
2. ¿Se tocó solo el alcance acordado?
3. ¿Se evitó romper producción?
4. ¿Se mantuvo SSR dinámico en Home?
5. ¿Hay validación real?
6. ¿Hay commit?
7. ¿Hay push?
8. ¿Hay deploy verificado si aplica?
9. ¿Se actualizó `PROJECT_STATE.md`?
10. ¿Se actualizó `EXECUTION_PLAN.md` si cambió el estado del plan?

## Definición de done
Una tarea NO está terminada si falta cualquiera de estas:
- código
- validación
- evidencia
- commit
- push
- deploy verificado si aplica
